# Claude Code 项目约束指令

## 基本要求

- **Python 版本**: 使用 Python 3.13
- **包管理**: 使用 `uv`

## 核心原则

- 绝不重复造轮子：实现功能前先搜索现有 PyPI 库或内置模块
- 只有确认无合适第三方库时才允许自己编写代码
- 优先使用 Python 标准库，但复杂逻辑必须用第三方库

## 实现流程

实现任何功能必须走"三步法"：
1. 搜索与推荐：搜索至少 2 个知名库，提供简介、下载量、更新时间、优缺点，推荐一个并说明理由
2. 等待确认：询问用户"是否使用 [推荐库名] 来实现？"
3. 实现：用户同意后，使用 `uv` 安装，提供最小可用代码示例

## 技术栈要求

- **包管理**: 必须使用 `uv`
- **Python 版本**: 推荐 3.13（除非有兼容性要求）
- **类型检查**: mypy 或 pyright

## VSCode LSP 配置

项目已配置以下 VSCode 扩展和工具：

- **Pylance**: 微软的 Python LSP，提供类型检查和智能提示
- **配置文件**:
  - `.vscode/settings.json`: VSCode 工作区配置
  - `.vscode/extensions.json`: 推荐扩展列表

使用方式：
1. 安装 VSCode 扩展：Python、Pylance
2. 严格类型检查模式

## 禁止行为

- 直接输出明显可由标准库或第三方库实现的代码
- 说"我们可以自己写一个简单的..."来实现通用功能
- 手写连接池、线程池、重试装饰器等
- 忽略项目已有依赖推荐同类型其他库
- 给出 `pip install` 或 `poetry add` 命令

## 例外情况

以下情况允许自己实现：
- 需求极其简单，引入库比手写更复杂
- 用户明确要求"不使用第三方库，只用标准库"
- 功能是项目业务核心逻辑，且 PyPI 上没有合适的库

## 会话启动规则

每次会话开始时，必须：
1. 读取 `CLAUDE.md`（本文件）
2. 读取 `REQUIREMENTS.md`（需求文档）
3. 理解项目当前状态、已实现功能和未来规划

## 工作原则

- 不要没有用户的明确指令，而自作主张的回退或前进你的工作。
